    <article class="markdown-body entry-content" itemprop="text"><h1><a id="user-content-project-distributed-social-networking" class="anchor" href="#project-distributed-social-networking" aria-hidden="true"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Project: Distributed Social Networking</h1>
<h2><a id="user-content-description" class="anchor" href="#description" aria-hidden="true"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Description</h2>
<p>The web is fundamentally interconnected and peer to peer. There’s
  no really great reason why we should all use facebook.com or
  google+ or myspace or something like that. If these social networks
  came up with an API you could probably link between them and use
  the social network you wanted. Furthermore you might gain some
  autonomy.</p>
<p>Thus in the spirit of diaspora <a href="https://diasporafoundation.org/">https://diasporafoundation.org/</a> we
  want to build something like diaspora but far far simpler.</p>
<p>This blogging/social network platform will allow the importing of
  other sources of posts (github, twitter, etc.) as well allow the
  distributing sharing of posts and content.</p>
<p>An author sitting on one server can aggregate the posts of their
  friends on other servers.</p>
<p>We won’t be adding much in the way of encryption or security to
  this platform. We’re keeping it simple and restful.</p>
<p>Choose at least 2 other groups to work with</p>

<h2><a id="user-content-user-stories" class="anchor" href="#user-stories" aria-hidden="true"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>User Stories</h2>
<ul>
  <li>As an author I want to make posts.</li>
  <li>As an author, posts I create can link to images.</li>
  <li>As a server admin, images can be hosted on my server.</li>
  <li>As an author, posts I create be private to me</li>
  <li>As an author, posts I create be private to another author</li>
  <li>As an author, posts I create be private to my friends</li>
  <li>As an author, posts I create be private to friends of friends</li>
  <li>As an author, posts I create be private to only friends on my host</li>
  <li>As an author, posts I create can be public</li>
  <li>As an author, posts I make can be in simple plain text</li>
  <li>As an author, posts I make can be in markdown (commonMark is good)</li>
  <li>As an author, I want a consistent identity per server</li>
  <li>As a server admin, I want to host multiple authors on my server</li>
  <li>As a server admin, I want to share or not share posts with users
    on other servers.</li>
  <li>As a server admin, I want to share or not share images with users
    on other servers.</li>
  <li>As an author, I want to pull in my github activity to my “stream”</li>
  <li>As an author, I want to post posts to my “stream”</li>
  <li>As an author, I want to delete my own posts.</li>
  <li>As an author, I want to befriend local authors</li>
  <li>As an author, I want to befriend remote authors</li>
  <li>As an author, I want to feel safe about sharing images and
    posts with my friends – images should not publicly accessible
    without authentication.</li>
  <li>As an author, I want un-befriend local and remote authors</li>
  <li>As an author, I want to be able to use my web-browser to manage
    my profile</li>
  <li>As an author, I want to be able to use my web-browser to manage/author
    my posts</li>
  <li>As a server admin, I want to be able add, modify, and remove
    authors.</li>
  <li>As a server admin, I want to be able allow users to sign up but
    require my OK to finally be on my server</li>
  <li>As a server admin, I don’t want to do heavy setup to get the
    posts of my author’s friends.</li>
  <li>As a server admin, I want a restful interface for most operations</li>
  <li>As an author, other authors cannot modify my data</li>
  <li>As an author, I want to comment on posts that I can access</li>
  <li>As an author, my server will know about my friends</li>
  <li>As an author, When I befriend someone it follows them, only when
    the other authors befriends me do I count as a real friend.</li>
  <li>As an author, I want to know if I have friend requests.</li>
  <li>As an author I should be able to browse the public posts of everyone</li>
  <li>As an author I should be able to browse the posts of others depending on my status</li>
  <li>As a server admin, I want to be able to add nodes to share with</li>
  <li>As a server admin, I want to be able to remove nodes and stop
    sharing with them.</li>
  <li>As a server admin, I can limit nodes connecting to me via
    authentication.</li>
  <li>As a server admin, node to node connections can be authenticated
    with HTTP Basic Auth</li>
  <li>As a server admin, I can disable the node to node interfaces for
    connections that are not authenticated!</li>
</ul>

<h2><a id="user-content-requirements" class="anchor" href="#requirements" aria-hidden="true"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Requirements</h2>

  <li>[ ] Prove your project by connecting with at least 2 other
